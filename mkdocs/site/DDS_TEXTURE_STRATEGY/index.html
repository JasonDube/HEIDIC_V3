    Compression Formats¶

    BC7 (Albedo, Diffuse, Base Color)¶

    • Use for: RGB/RGBA textures (color maps)
    • Quality: Excellent (near-lossless)
    • Size: ~1-2MB for 2048√ó2048
    • Load: 0.8-2ms

    BC5 (Normal Maps)¶

    • Use for: RG textures (normal maps, height maps)
    • Quality: Excellent (2-channel)
    • Size: ~1MB for 2048√ó2048
    • Load: 0.8-2ms

    R8 (Grayscale)¶

    • Use for: Single-channel textures (roughness, metalness, emissive)
    • Quality: Lossless (8-bit grayscale)
    • Size: ~0.5MB for 2048√ó2048
    • Load: <1ms

    Build Pipeline Integration¶

    H_SCRIBE Auto-Conversion¶

    On Build: 1. Scan for PNG files in textures/source/ or textures/ 2. Convert each PNG to DDS: - Albedo/diffuse ‚Üí BC7 - Normal maps ‚Üí BC5 - Grayscale ‚Üí R8 3. Output to textures/ (compiled assets) 4. Reference compiled .dds in generated code

    Conversion Tool: - Windows: texconv.exe (DirectXTex) - Cross-platform: toktx (KTX-Software) ‚Üí KTX2 - Custom: Write minimal DDS writer (if zero dependencies needed)

    Build Script: 

    # In H_SCRIBE build process
def convert_textures():
    for png_file in find_png_files("textures/source/"):
        dds_file = png_file.replace(".png", ".dds")

        # Detect texture type from name
        if "normal" in png_file.lower():
            format = "BC5"  # RG for normals
        elif any(x in png_file.lower() for x in ["rough", "metal", "emissive"]):
            format = "R8"   # Grayscale
        else:
            format = "BC7"  # RGB/RGBA

        # Convert PNG ‚Üí DDS
        run_converter(png_file, dds_file, format)

    Runtime vs Source Assets¶

    Runtime (Production)¶

    • ‚úÖ Load .dds files (fast, GPU-ready)
    • ‚úÖ 1.1ms load time
    • ‚úÖ Zero-copy upload
    • ‚úÖ Small memory footprint

    Development/Editor¶

    • ‚úÖ Edit .png files (human-readable)
    • ‚úÖ Auto-convert to .dds on build
    • ‚úÖ LLM can read/edit PNG metadata
